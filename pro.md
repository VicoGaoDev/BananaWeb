# PRO 更新记录

## 本次新增

### 场景积分动态配置
- 将任务积分从前后端硬编码改为按 `external_api_scene_bindings.credit_cost` 动态读取。
- 文生图按所选模型场景的单价按张扣费。
- 局部重绘按 `inpaint` 场景固定积分按次扣费。
- 提示词反推按 `prompt_reverse` 场景固定积分按次扣费。
- 超级管理员免积分逻辑保持不变。

### 接口管理页
- 场景绑定区新增“消耗积分”配置项。
- 超级管理员可在同一行同时维护调用场景的绑定接口和积分值。
- 接口配置与场景绑定继续保持解耦，积分配置挂在场景层，不挂在接口层。

### 前端展示联动
- 生成页“开始生成”按钮按所选模型实时展示积分。
- 提示词反推按钮按 `prompt_reverse` 场景实时展示积分。
- 前端余额校验与后端实际扣费改为读取同一套场景积分配置。
- 新增公开接口 `/api/config/task-scenes` 供前端读取场景积分与展示信息。

## 默认兼容值
- `banana` / `banana2` / `banana_pro` / `banana_pro_plus` 默认 `4` 积分 / 张
- `inpaint` 默认 `4` 积分 / 次
- `prompt_reverse` 默认 `1` 积分 / 次

## 已完成验证
- `python3 -m compileall backend/app`
- `npm run build`

## 补充更新

### 历史记录筛选
- 历史记录页新增按任务类型筛选，支持 `生图` 与 `局部重绘` 两类。
- 历史记录页新增按模型筛选。
- 历史记录页新增按任务状态筛选，支持 `等待中`、`处理中`、`成功`、`失败`。
- 历史记录页新增按提示词关键词筛选。
- 历史记录页新增按时间范围筛选。
- 用户历史接口 `/api/history` 新增 `mode`、`model`、`prompt`、`status`、`start_date`、`end_date` 查询参数。
- 历史记录卡片补充展示任务类型，便于区分来源。

### 历史记录重构
- 用户历史记录从按任务展示改为按结果图展示，每张结果图单独一张卡片。
- 历史页顶部总览压缩为轻量信息条，筛选栏改为内联工具条，不再单独占卡片。
- 卡片主视图仅展示结果图、状态、类型、时间，提示词与参数移入详情弹窗。
- 新增单图删除接口 `/api/images/:id`，删除最后一张时会自动清理空任务。
- 局部重绘历史点击“重新编辑”时会直接回到局部重绘 tab，并回填原图与提示词。
